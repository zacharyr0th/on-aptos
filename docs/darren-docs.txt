## Gas Usage

Grab an account‚Äôs transaction history per specified time frame.

```
https://api.mainnet.aptoslabs.com/v1/analytics/gas/usage?gas_payer_address=<string>&start_unix_secs=<int>&end_unix_secs=<int>
```

<aside>
üîó

Try this endpoint in your [browser](https://api.mainnet.aptoslabs.com/v1/analytics/gas/usage?gas_payer_address=0x0000009610c185cf2d3aa4ab7a6e352730b386cc5c9268671c606826253e740c&start_unix_secs=1704067200&end_unix_secs=1735689599)

</aside>

- Request
    
    
    | Query Parameter | Description |
    | --- | --- |
    | account_address | the `string` account address. |
    | start_utc_unix_secs | the (inclusive) start UTC time of the range query `integer` in unix timestamp in seconds  |
    | end_utc_unix_secs | the (inclusive) end UTC time of the range query `integer` in unix timestamp in seconds  |
    | bucket_granularity | an optional `string` that aggregates usage into buckets. Uses the previous day‚Äôs closing price for usage in USD to be consistent with the non-aggregated query. possible granularities: `1s`, `1m`, `5m`, `15m` , `1h`, `6h`, `1d`, `7d` |
    | group_by_entry_function` | an optional `bool` that aggregates gas usage into entry function type. |
- Sample Responses
    - Successful Response
        
        ```json
        {
          "data": [
            {
              "total_gas_used_octas": 11136916721,
              "total_gas_used_usd": 1135.890586339156
            }
          ],
          "error": null,
          "message": "Request completed successfully",
          "statistics": {
            "bytes_read": 73796498,
            "elapsed": 0.087801081,
            "rows_read": 791375
          },
          "status": "success"
        }
        ```
        
    - Failed Response
        
        ```json
        {
          "status": "error",
          "message": "Internal Server Error",
          "data": null,
          "error": "Failed to send request to ClickHouse: Timeout"
        }
        ```
        

## Single Token Pricing History

The price history for an individual token.

```
https://api.mainnet.aptoslabs.com/v1/analytics/token/historical_prices?address=<asset_address>&lookback=<hour|day|week|month>
```

<aside>
‚òù

Some non-major tokens unavailable on CMC or CG only have history starting on 2/11

</aside>

<aside>
üîó

Try this endpoint in your [browser](https://api.mainnet.aptoslabs.com/v1/analytics/token/historical_prices?address=0x1::aptos_coin::AptosCoin&lookback=month) 

</aside>

- Request
    
    
    | Query Parameter | Description |
    | --- | --- |
    | **address** | the `string` representing the contract address for the token/FA. `APT`, `BTC`, `ETH` strings are supported.  |
    | **lookback** | `string: [day, week, month, year, all]` that determines the lookback window. `day` , `week` , `month` , `year`, `all`, will return 5min, 1hr, 4hr, daily, daily closes respectively.  |
    | limit | optional pagination parameter |
    | offset | optional offset parameter |
    | downsample_to | optional downsample param. If original query returns some `N` datapoints, setting downsample_to= `n` will return `downsampled_data[0]` = `orig_data[0]`, `downsampled_data[n]` = `orig_data[N]` and middle datapoints uniformly distributed based on the ratio of `n` to `N`.  |
- Sample Responses
    - Successful Response
        
        ```json
        {
        	"data":[
        		{
        	    "bucketed_timestamp_minutes_utc": "2025-02-11 16:05:00",
              "price_usd": 94889.0965771
        		},
        		{
              "bucketed_timestamp_minutes_utc": "2025-02-11 16:00:00",
              "price_usd": 97802.73433672
        		},
        		...
        	],
        	"error":null,
        	"message":"Request completed successfully",
        	"status":"success"
        }
        ```
        
    - Failed Response
        
        ```json
        {
          "status": "error",
          "message": "Internal Server Error",
          "data": null,
          "error": "Failed to send request to ClickHouse: Timeout"
        }
        ```
        

## Latest Token Price

Grab the latest price of a token 

```
https://api.mainnet.aptoslabs.com/v1/analytics/token/latest_price?address=<asset_address>&date_utc=<yyyy-mm-dd>
```

<aside>
‚òù

Optionally add `&date_utc=<yyyy-mm-dd>` to get the opening price for a particular UTC date

Asset symbols are also supported for majors (APT, BTC, ETH)

</aside>

<aside>
üîó

Try this endpoint in your [browser](https://api.mainnet.aptoslabs.com/v1/analytics/token/latest_price?address=0x1::aptos_coin::AptosCoin&date_utc=2025-02-18)

</aside>

- Request
    
    
    | Query Parameter | Description |
    | --- | --- |
    | asset_address | the `string` asset address. |
    | date_utc | optional, the `YYYY-MM-DD` specify the opening price for a date. If not provided, the endpoint will return the latest price stored.  |

## Top Price Percentage Changes

Grab the greatest token price percentage changes in the last day/week/month.  

```
https://api.mainnet.aptoslabs.com/v1/analytics/token/top_price_changes?lookback=<hour|day|week|month>&limit=<int>&only_emoji=<bool>&gainers=<bool>
```

<aside>
üîó

Try this endpoint in your [browser](https://api.mainnet.aptoslabs.com/v1/analytics/token/top_price_changes?lookback=day&limit=5)

</aside>

- Request
    
    
    | Query Parameter | Description |
    | --- | --- |
    | lookback | the `string: hour, day, week, month` for time horizon on price changes.  |
    | limit | the `int` limiting how many rows you get |
    | only_emoji | optional `bool` for if you only want emoji coins in the response. if `false`, the response will include a mix of emoji coins and other FA/coins.
    defaults to `false`. |
    | gainers | optional `bool` . if set to `true`, endpoint will return top N percentage price ***gainers*** in the lookback period. `false` will return top N ***losers*** in the same period.
    defaults to `true`.  |
